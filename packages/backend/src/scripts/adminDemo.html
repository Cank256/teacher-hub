üìã System Overview

Database Schema

The role system includes the following database tables:

  • users: Extended with role column
  • permissions: Defines all available permissions
  • role_permissions: Maps roles to permissions
  • user_permissions: Individual user permission overrides
  • role_audit_log: Tracks role changes

API Endpoints

New role management endpoints:

  • GET /api/roles/permissions - Get all permissions
  • GET /api/roles/users/:userId - Get user role and permissions
  • PUT /api/roles/users/:userId/role - Change user role
  • POST /api/roles/users/:userId/permissions/:permission/grant - Grant permission
  • GET /api/roles/admin-users - Get all admin users
  • GET /api/roles/check/:permission - Check if user has permission
